## トリガー問題の診断

### アンダートリガー（起動しない）のサイン
- スキルが関連クエリで起動しない
- ユーザーが手動で有効化している
- 「いつ使うのか」という質問がある

**対処**: description に具体的なキーワードとトリガーフレーズを追加する。

デバッグ方法: Claudeに「〇〇スキルをいつ使いますか？」と質問する。Claudeが description を引用して回答するので、不足しているフレーズを確認する。

### オーバートリガー（起動しすぎ）のサイン
- 無関係なクエリでスキルが起動する
- ユーザーがスキルを無効化している
- 目的についての混乱

**対処**:
1. ネガティブトリガーを追加する:
   ```yaml
   description: ...（通常の説明）。CSVの単純な探索には使用しない（data-vizスキルを使用すること）。
   ```
2. より具体的にする（「ドキュメントを処理する」→「PDF法的文書を契約レビューのために処理する」）
3. スコープを明確にする
