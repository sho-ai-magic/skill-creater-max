# Anthropic公式スキルガイド 要点まとめ

## スキルとは

スキルは特定タスク・ワークフローをClaudeに教えるフォルダ一式。
毎回説明し直す手間をなくし、一度教えれば繰り返し使える。
Claude.ai / Claude Code / API で同一スキルが動作する。

---

## 3段階Progressive Disclosure（プログレッシブ開示）

| レベル | 内容 | 読み込みタイミング |
|--------|------|------------------|
| YAMLフロントマター | スキルの存在を認識させる | 常にシステムプロンプトに読み込まれる |
| SKILL.md 本文 | 詳細な指示 | 関連タスク時のみ |
| リンクファイル（references/等） | 参照ドキュメント | 必要時のみ |

→ トークン消費を最小化しつつ専門機能を発揮する。

---

## ユースケース3分類

### カテゴリ1: ドキュメント・アセット作成

用途: 一貫した品質のドキュメント、デザイン、コード、プレゼン等を生成する。

主な技術:
- スタイルガイドやブランド基準を埋め込む
- 一貫した出力のためのテンプレート構造
- 完成前の品質チェックリスト
- 外部ツール不要（Claudeの組み込み機能のみ）

### カテゴリ2: ワークフロー自動化

用途: 一貫した手順が必要な多段階プロセス、複数MCPサーバーにまたがる調整。

主な技術:
- バリデーションゲートを含むステップ別ワークフロー
- 共通構造のテンプレート
- レビューと改善提案の内蔵
- 反復改善ループ

### カテゴリ3: MCPエンハンスメント

用途: MCPサーバーが提供するツールアクセスにワークフロー知識を上乗せする。

主な技術:
- 複数のMCP呼び出しを順番に調整
- ドメイン専門知識を埋め込む
- ユーザーが都度指定する必要のあるコンテキストを提供
- よくあるMCPエラーのハンドリング

## 成功基準（テスト指標）

### 定量指標

- **スキルトリガー率**: 関連クエリ10〜20件中90%以上でスキルが自動起動する
- **ツール呼び出し回数**: スキルなしと比較して削減されている
- **APIエラー数**: ワークフロー1回あたり0件

### 定性指標

- ユーザーが次のステップを都度促す必要がない
- ユーザーが訂正を入れずにワークフローが完了する
- 新規ユーザーが最小限の案内で初回から成功できる

---

## テスト3段階

### 1. トリガーテスト

目標: 適切なタイミングでスキルが起動すること。

チェック:
- ✅ 明示的なタスクでトリガーされる
- ✅ 言い換えた表現でトリガーされる
- ❌ 無関係なトピックではトリガーされない

デバッグ: Claudeに「〇〇スキルをいつ使いますか？」と質問するとdescriptionを引用して答える。回答を見て description を調整する。

### 2. 機能テスト

目標: 正しい出力が生成されること。

チェック:
- 有効な出力が生成される
- APIコールが成功する
- エラーハンドリングが機能する
- エッジケースに対応できる

### 3. パフォーマンス比較

目標: スキルが結果を改善することを証明する。

比較項目:
- スキルなし: ユーザーが毎回指示、多くの往復、失敗したAPIコール
- スキルあり: 自動ワークフロー実行、少ない確認質問、エラー0件

## 配布方法

### 個人ユーザー向け

1. スキルフォルダをダウンロード
2. フォルダを ZIP 圧縮
3. Claude.ai: Settings > Capabilities > Skills からアップロード
4. または Claude Code のスキルディレクトリに配置

### 組織向け（管理者）

- ワークスペース全体にスキルを展開（2025年12月対応）
- 自動更新・一元管理

### API向け

- `/v1/skills` エンドポイント
- Messages API リクエストの `container.skills` パラメータ
- Code Execution Tool beta が必要

### 推奨アプローチ

1. GitHub に パブリックリポジトリで公開（リポジトリルートに README.md）
2. MCPドキュメントからスキルへのリンクを追加
3. インストールガイドを作成
